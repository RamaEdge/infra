  <action>
Add nodeSelector and topologySpreadConstraints to the top-level values block in forgejo/helmrelease.yaml. Place the new keys BEFORE the existing persistence key so they appear at the top of values.

The Forgejo chart uses the Forgejo app deployment label app.kubernetes.io/name: forgejo for its pod selector. Use this in the topologySpreadConstraints labelSelector.

Add these two blocks to values:
```yaml
values:
  nodeSelector:
    node-role: apps

  topologySpreadConstraints:
    - maxSkew: 1
      topologyKey: kubernetes.io/hostname
      whenUnsatisfiable: ScheduleAnyway
      labelSelector:
        matchLabels:
          app.kubernetes.io/name: forgejo

  persistence:
    # ... existing unchanged ...
```

Use ScheduleAnyway (not DoNotSchedule) — Forgejo is a single-replica deployment so a hard constraint would block scheduling if the apps nodes have any spread imbalance.

All existing values (persistence, ingress, gitea, resources) must remain unchanged.
  </action>

All existing values (persistence, ingress, gitea, resources) are unchanged.</done>
